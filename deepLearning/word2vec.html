1. 跳字模型

跳字模型用于计算给定中心词计算背景词的条件概率，在跳字模型中每个词被表示成两个ddd维向量，用来计算条件概率。假设这个词在词典中的索引为iii，当它为中心词时向量表示为νi∈Rd\nu_i\in R^dνi​∈Rd，而为背景词是向量表示为μi∈Rd\mu_i\in R^dμi​∈Rd。

设中心词wcw_cwc​在词典中的索引为ccc，而背景词wow_owo​在词典中的索引为ooo，给定中心词生成背景词的条件概率可以通过对向量内积做softmax运算而得到：

P(wo∣wc)=exp⁡(μoTνc)∑i∈ϑexp⁡(μiTνc)P(w_o|w_c) =\frac{\exp(\mu_o^T\nu_c)}{\sum\limits_{i \in \vartheta} \exp(\mu_i^T\nu_c)}P(wo​∣wc​)=i∈ϑ∑​exp(μiT​νc​)exp(μoT​νc​)​

其中词典索引ϑ={0,1,…,∣ϑ∣−1}\vartheta=\{0, 1, \dots, |\vartheta|-1\}ϑ={0,1,…,∣ϑ∣−1}。

似然函数

假设给定一个长度为TTT的文本序列，设时间步ttt的词为w(t)w^{(t)}w(t)。假设给定中心词的情况下背景词的生成相互独立，当背景窗口大小为mmm时，跳字模型的似然函数，即给定任一中心词生成所有背景词的概率：

2. 连续词袋模型

设νi∈Rd,μi∈Rd\nu_i \in R^d, \mu_i\in R^dνi​∈Rd,μi​∈Rd，分别表示词典中索引为iii的背景词和中心词的向量（注意符号和跳字模型中的相反）。设中心词wcw_cwc​在词典中的索引为ccc，背景词wo1,wo2,…,wo2mw_{o_1},w_{o_2},\ldots,w_{o_{2m}}wo1​​,wo2​​,…,wo2m​​在词典中的索引为o1,o2,…,o2mo_1, o_2, \ldots, o_{2m}o1​,o2​,…,o2m​，那么给定背景词生成中心词的条件概率为：

P(wc∣wo1,wo2,…,wo2m)=exp⁡(12mμcT(νo1+νo2+⋯+νo2m))∑i∈ϑexp⁡(12mμiT(νo1+νo2+⋯+νo2m))P(w_c|w_{o_1}, w_{o_2}, \ldots, w_{o_{2m}})=\frac{\exp\left(\frac{1}{2m}\mu_c^T(\nu_{o_1}+\nu_{o_2}+\cdots+\nu_{o_{2m}})\right)}{\sum\limits_{i\in\vartheta}\exp\left(\frac{1}{2m}\mu_i^T(\nu_{o_1}+\nu_{o_2}+\cdots+\nu_{o_{2m}}) \right)}P(wc​∣wo1​​,wo2​​,…,wo2m​​)=i∈ϑ∑​exp(2m1​μiT​(νo1​​+νo2​​+⋯+νo2m​​))exp(2m1​μcT​(νo1​​+νo2​​+⋯+νo2m​​))​

3. 近似训练

跳字模型的核心在于使用softmax运算得到给定中心词wcw_cwc​来生成背景词wow_owo​的条件概率。

P(wo∣wc)=exp⁡(μoTνc)∑i∈ϑexp⁡(μiTνc)P(w_o|w_c) = \frac{\exp(\mu_o^T\nu_c)}{\sum\limits_{i\in\vartheta}\exp(\mu_i^T\nu_c)}P(wo​∣wc​)=i∈ϑ∑​exp(μiT​νc​)exp(μoT​νc​)​

该条件函数对应的损失函数：

−log⁡P(wo∣wc)=−μoTνc+log⁡(∑i∈ϑexp⁡(μiTνc))-\log P(w_o|w_c) = -\mu_o^T\nu_c + \log\left( \sum\limits_{i\in\vartheta}\exp(\mu_i^T\nu_c) \right)−logP(wo​∣wc​)=−μoT​νc​+log(i∈ϑ∑​exp(μiT​νc​))

由于softmax运算考虑了背景词可能是词典ϑ\varthetaϑ中的任意词，以上损失函数包含了词典大小数目的项的累加，在上一节中我们看到，不论是跳字模型还是连续词袋模型，由于条件概率使用了softmax运算，每一步的梯度计算都包含了词典大小数目的项的累加。对于包含上百万词的较大词典，每次梯度计算的开销过大。为了降低计算复杂度，引入了近似训练方法，包括负采样和层序softmax。由于跳字模型和连续词袋模型类似，因此下面仅以跳字模型为例。

3.1 负采样

负采样修改了原来的目标函数，给定中心词wcw_cwc​的一个背景窗口，我们把背景词wow_owo​出现在该窗口看作是一个事件，并将该事件的概率计算为：

P(D=1∣wc,wo)=σ(μoTνc)P(D=1|w_c,w_o) = \sigma(\mu_o^T\nu_c)P(D=1∣wc​,wo​)=σ(μoT​νc​)

其中σ\sigmaσ函数与sigmoid激活函数的定义相同：

σ(x)=11+exp⁡(−x)\sigma(x) = \frac{1}{1+\exp(-x)}σ(x)=1+exp(−x)1​

我们先考虑最大化文本序列中所有该时间的联合概率来训练词向量，具体的说，给定一个长度为TTT的文本序列，设时间步ttt的中心词为w(t)w^{(t)}w(t)且背景窗口的大小为mmm，考虑最大化联合概率：

∏t=1T∏−m≤j≤m,j≠0P(D=1∣w(t),w(t+j))\prod\limits_{t=1}^T \prod\limits_{-m\leq j \leq m, j\neq0} P\left(D=1|w^{(t)},w^{(t+j)}\right)t=1∏T​−m≤j≤m,j≠0∏​P(D=1∣w(t),w(t+j))

然而，以上模型中包含的事件仅考虑了正类样本，这导致所有词向量相等且值为无穷大时，以上的联合概率才被最大化为1。很明显这样的词毫无意义。

负采样通过采样并添加负类样本使目标函数更有意义。设背景词wow_owo​出现在中心词wcw_cwc​的一个背景窗口为事件PPP，我们根据分布P(w)P(w)P(w)采样KKK个未出现在该背景窗口中的词，即噪声词。设噪声词wk,k=1,2,…,Kw_k, k=1,2,\ldots, Kwk​,k=1,2,…,K不出现在中心词wcw_cwc​的该背景窗口为事件NkN_kNk​。假设同时含有正类样本和负类样本的事件P,N1,N2,…,NKP,N_1,N_2,\ldots, N_KP,N1​,N2​,…,NK​相互独立，负采样将以上需要最大化的仅考虑正类样本的联合概率改写为：

∏t=1T∏−m≤j≤m,j≠0P(w(t+j)∣w(t))\prod\limits_{t=1}^T \prod\limits_{-m\leq j \leq m, j\neq0} P\left(w^{(t+j)}|w^{(t)}\right)t=1∏T​−m≤j≤m,j≠0∏​P(w(t+j)∣w(t))

其中条件概率被近似表示为：

P(w(t+j)∣w(t))=P(D=1∣w(t),w(t+j))∏k=1,wk∼P(w)KP(D=0∣w(t),wk)P\left(w^{(t+j)}|w^{(t)}\right) = P(D=1|w^{(t)},w^{(t+j)}) \prod\limits_{k=1,w_k\sim P(w)}^K P(D=0|w^{(t)},w_k)P(w(t+j)∣w(t))=P(D=1∣w(t),w(t+j))k=1,wk​∼P(w)∏K​P(D=0∣w(t),wk​)

设文本序列中时间步ttt的词w(t)w^{(t)}w(t)在词典中的索引为iti_tit​，噪声词wkw_kwk​在词典中的索引为hkh_khk​。有关以上条件概率的对数损失为：

−log⁡P(w(t+j)∣w(t))=−log⁡P(D=1∣w(t),w(t+j))−∑k=1,wk∼P(w)Klog⁡P(D=0∣w(t),wk)-\log P(w^{(t+j)}|w^{(t)}) = -\log P(D=1|w^{(t)},w^{(t+j)}) - \sum\limits_{k=1,w_k\sim P(w)}^K \log P(D=0|w^{(t)},w_k)−logP(w(t+j)∣w(t))=−logP(D=1∣w(t),w(t+j))−k=1,wk​∼P(w)∑K​logP(D=0∣w(t),wk​)
=−log⁡σ(μit+jTνit)−∑k=1,wk∼P(w)Klog⁡(1−σ(μhkTνit))= -\log\sigma\left(\mu_{i_{t+j}}^T\nu_{i_t}\right) - \sum\limits_{k=1,w_k\sim P(w)}^K\log \left(1-\sigma\left(\mu_{h_k}^T\nu_{i_t}\right)\right)=−logσ(μit+j​T​νit​​)−k=1,wk​∼P(w)∑K​log(1−σ(μhk​T​νit​​))

现在，训练中每一步的梯度计算开销不再与词典大小相关，而与KKK线性相关。当KKK取较小的常数时，负采样在每一步的梯度计算开销较小。

 # Step 3: Function to generate a training batch for the skip-gram model.
def generate_batch(batch_size, num_skips, skip_window):
    global data_index
    assert batch_size % num_skips == 0
    assert num_skips <= 2 * skip_window
    batch = np.ndarray(shape=(batch_size), dtype=np.int32)
    labels = np.ndarray(shape=(batch_size, 1), dtype=np.int32)
    span = 2 * skip_window + 1  # [ skip_window target skip_window ]
    buffer = collections.deque(maxlen=span)  # pylint: disable=redefined-builtin
    if data_index + span > len(data):
      data_index = 0
    buffer.extend(data[data_index:data_index + span])
    data_index += span
    for i in range(batch_size // num_skips):
      context_words = [w for w in range(span) if w != skip_window]
      words_to_use = random.sample(context_words, num_skips)
      for j, context_word in enumerate(words_to_use):
        batch[i * num_skips + j] = buffer[skip_window]
        labels[i * num_skips + j, 0] = buffer[context_word]
      if data_index == len(data):
        buffer.extend(data[0:span])
        data_index = span
      else:
        buffer.append(data[data_index])
        data_index += 1
    # Backtrack a little bit to avoid skipping words in the end of a batch
    data_index = (data_index + len(data) - span) % len(data)
    return batch, labels
